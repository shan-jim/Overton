How the Data Powers Quoting

ProfitKey stores every clock-in and clock-out as a discrete row in LABOR_TICKETS — the summary hours shown on the report are calculated from this raw transaction history. An ODBC connection pulls that atomic detail directly from the ProfitKey SQL Server, read-only, without touching shop floor operations. That extract lands in a separate analytical database where the quoting team can slice actual hours by work center, employee, or operation type — turning real production history into defensible quote estimates.
